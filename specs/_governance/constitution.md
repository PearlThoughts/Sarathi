# Sarathi Constitution

## Status

Accepted for the initial platform foundation.

## Principles

1. Sarathi is human-guided, policy-bounded, and evidence-gated.
2. Better Auth provides identity and membership; Sarathi policy owns sensitivity and action boundaries.
3. YAML overlays describe boundary intent but do not enforce security by themselves.
4. Authorization must run before retrieval, tool invocation, and model egress.
5. Repository docs explain implementation and architecture; product strategy lives in SenG-Vault.
